Question: Q1: Terraform declarative hai ya imperative? Iska kya fayda hai?
Answer: A: Terraform declarative hai — aap sirf "kya chahiye" batate hain, "kaise banana hai" nahi. Isse configuration simple hoti hai, aur Terraform automatically best path choose karta hai to reach desired state.
Question: Q2: Terraform state file kya hota hai? Iske bina kya ho sakta hai?
Answer: A: Terraform state file (terraform.tfstate) me infrastructure ka actual current state store hota hai. Iske bina Terraform ko nahi pata hoga ki kya changes apply karne hain, aur woh har baar infrastructure ko naya banane ki koshish karega.
Question: Q3: Terraform plan aur apply me kya difference hai?
Answer: A: terraform plan: Dry run, sirf dikhata hai kya hone wala hai. terraform apply: Actual infrastructure me changes karta hai. 🗂 State Management
Question: Q4: State file secure kaise rakhen?
Answer: A: Remote backends use karein (S3 + DynamoDB for locking, Azure Blob + state locking) encrypt = true option Access control via IAM
Question: Q5: Terraform Locking kya hota hai?
Answer: A: Locking ensure karta hai ki ek hi time me sirf ek user state file modify kare. Jaise AWS me S3 + DynamoDB locking ya Azure me Blob lease-based locking.
Question: Q6: Terraform Refresh kya karta hai?

Answer: Here’s a breakdown of answers and explanations to the PMYgroup interview questions you listed. These are concise but technical enough to support you during interviews: ---
Question: 1. What is Azure Functions?
Answer: Azure Functions is a serverless compute service that lets you run small pieces of code (functions) without worrying about infrastructure. It scales automatically and you only pay for the time your code runs. ---
Question: 2. What is NAT and Application Gateway?
Answer: NAT (Network Address Translation): Translates private IPs to a public IP, enabling outbound internet access for VMs in private subnets. Azure Application Gateway: A layer 7 load balancer with features like SSL termination, Web Application Firewall (WAF), and URL-based routing. ---
Question: 3. What is sidecar container?
Answer: A sidecar container is a helper container that runs alongside the main container in the same Pod. It's commonly used for logging, proxying (e.g., Istio Envoy), or configuration syncs. ---
Question: 4. Can we deploy more than one container in a Pod?
Answer: Yes, a Pod can run multiple containers that share the same network namespace and storage volumes. This is useful for tightly coupled applications (e.g., app + logging agent). ---
Question: 5. What is taint? Why is it used?
Answer: A taint in Kubernetes prevents Pods from being scheduled on a Node unless the Pod tolerates that taint. It ensures that only specific Pods run on specific Nodes. ---
Question: 6. What is headless service?
Answer: A headless service in Kubernetes has no ClusterIP and allows clients to directly discover and connect to individual Pod IPs (DNS returns multiple A records). ---
Question: 7. What is deployment set?
Answer: Possibly a mix-up with Deployment or StatefulSet: Deployment manages stateless replicas. StatefulSet manages stateful applications with persistent identity and storage. ---
Question: 8. What is Docker Swarm?
Answer: Docker Swarm is Docker's native container orchestration tool, managing container clusters, offering service discovery, scaling, and load balancing. ---
Question: 9. What is ARG in Docker?
Answer: ARG is a Dockerfile instruction that defines build-time variables. Unlike ENV, ARG values are not available after the image is built. ---
Question: 10. How do you manage Terraform state files (multiple state files)?
Answer: Use remote backends like Azure Storage with state locking. Split resources by workspaces or modular state files (e.g., one per environment or team). Use terraform_remote_state data source to refer to outputs across state files. ---
Question: 11. Why use MySQL over other databases?
Answer: Open-source, widely supported. Great for structured data with relational models. ACID compliance ensures data integrity. Rich ecosystem with tools, connectors, and integrations. ---
Question: 12. How to manage a /16 CIDR with thousands of IPs?
Answer: Divide the range using subnetting (/24, /25 etc.). Use naming conventions and IPAM (IP Address Management) tools. Automate using Terraform/ARM and define clear IP allocation policies. ---
Question: 13. If 150 people are using the same backend, how do you handle/recognize them?
Answer: Use authentication tokens (JWT), headers, or API keys to identify users. Use load balancers with sticky sessions (if needed). Collect logs/metrics with user IDs tagged. ---
Question: 14. What is root module and child module in Terraform?
Answer: Root module: The main directory where Terraform commands are run. Child module: A reusable module invoked from the root or other modules using module blocks. ---
Question: 15. How to avoid hardcoding IPs in main.tf?
Answer: Use variables and pass values via .tfvars files or pipeline inputs. Store IPs in Key Vault, Parameter Store, or output from another module. ---
Question: 16. What is the use of top command in Linux?
Answer: top shows real-time system resource usage (CPU, memory, processes). Helps monitor and troubleshoot system performance. ---
Question: 17. What is StatefulSet in Kubernetes?
Answer: StatefulSet manages stateful applications. Ensures: Stable network identity (Pod names) Ordered startup, scaling, and termination Persistent storage with PVCs
